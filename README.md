# Software Engineering Project

This repository contains a Unity project using C#, designed for game development or interactive media.

## Project Structure

- **Assets/**: This folder is where all game assets are stored, including scripts (C# code), scenes, prefabs, textures, audio files, and other resources Unity uses during game development.
- **Packages/**: Contains configuration files and references for Unity package dependencies. This ensures all required packages are correctly set up when the project is opened.
- **ProjectSettings/**: Stores all Unity project settings (graphics, input, build, tags, etc.). These files define how Unity should build, operate, and run the project.
- **.gitignore**: Specifies files and folders to be excluded from version control. Typical for Unity projects, it prevents tracking of generated files, local settings, and user-specific configs.
- **.vsconfig**: Configuration for Visual Studio. It requests the 'Managed Game' workload, which is required for Unity/C# integration in Visual Studio.

## Requirements

- Unity (recommended version: check your Unity Hub for compatibility)
- Visual Studio (with the Managed Game workload)

## Getting Started

1. Clone this repository.
2. Open the project with Unity Hub by selecting the root folder.
3. Restore dependencies using Unity's package manager if prompted.
4. Open and edit scripts in Visual Studio for a seamless development experience.
